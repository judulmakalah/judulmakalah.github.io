1. PERBEDAAN ANTARA LINIER PROGRAMING METODE GRAFIK DENGAN LINIER PROGRAMING METODE SIMPLEKS. JELASKAN.​


Perbedaan metode grafik dan simpleks adalah metode grafik hanya sanggup dipakai untuk dua masalah saja. Sementara, metode simpleks sanggup menyelesaikan kendala lebih dari dua kendala, yang berarti metode ini mempunyai setidaknya tiga kendala didalamnya.

Pembahasan

Program linear merupakan suatu program yang memiliki fungsi menyelesaikan berbagai masalah yang berhubungan dengan suatu variabel dan memiliki batasan dengan pertidaksamaan linear. Program linear juga disebut dengan metode matematika yang berfungsi memecahkan masalah berkaitan dengan optimasi linear (maksimum/minimum). Persoalan yang berkaitan dengan program linier selalu terkait dengan fungsi objektif berdasarkan kondisi yang membatasinya. Dalam hal ini, optimasi berupa memaksimalkan atau meminimalkan fungsi objektif.Metode grafik merupakan metode yang digunakan untuk menyelesaikan permasalahan program linear yang sering dipakai, karena mudah dan tidak menghabiskan banyak waktu. Hanya saja, metode ini hanya bisa dipakai untuk menyelesaikan permasalahan program linear dengan dua variabel saja. Metode simpleks merupakan teknik penentuan solusi optimal yang dipakai pada pemrograman linier yang dilakukan dengan memeriksa titik ekstrim satu persatu dengan perhitungan interaktif. Sehingga, penentuan solusi optimal dengan metode simpleks tersebut dilakukan dengan tahap demi tahap yang kita kenal dengan iterasi.

Persoalan linear programing yang diselesaikan dengan cara metode grafik dapat diselesaikan juga dengan metode simpleks. Namun sebaliknya, persoalan yang hanya dapat diselesaikan dengan metode simpleks tidak akan bisa diselesaikan dengan metode grafik.

15. jelaskan perbedaan pemrograman server side dan client side programing


Client Side Scripting Client side scripting merupakan salah satu jenis bahasa pemrograman web yang proses pengolahannya dilakukan di sisi client. Proses penerjemahan atau pengolahannya dilakukan oleh browser sebagai clientnya, didalam web browser sudah disediakan atau terdapat library yang mampu menerjemahkan semua perintah dihalaman web yang menggunakan client side scripting. Library ini juga disebut web engine yang disetiap web browser memiliki web engine yang berbeda-beda sehingga tidak aneh jika tampilan layout web sering berbeda disetiap browser, untuk menghindari itu sebaiknya gunakan web browser yang telah disertifikasi oleh world wide web consortium(W3C). Contoh dari client side scripting ini antara lain HTML, XHTML, CSS, JavaScript, XML. sedangkaServer side scripting merupakan bahasa pemrograman web yang pengolahannya dilakukan dalam server, maksudnya ialah web server yang sudah telah terintegrasi oleh web engine. Disini peran web engine ialah memproses semua script yang ada termasuk kategori client side scripting. Web engine biasanya harus diinstal dalam komputer terlebih dahulu dari bagian terpisah web server.  Contoh dari server side scripting ialah Active server pages (ASP), PHP : Hypertext preprocessor (PHP), Java sever pages (JSP).

20. tuliskan 2 unsur pada teknik dynamic programing​


Jawaban:

Dalam teknik dynamic programming, terdapat dua unsur utama:

Overlapping Subproblems (Submasalah yang tumpang tindih): Dalam dynamic programming, masalah besar dibagi menjadi sejumlah submasalah yang lebih kecil. Unsur ini mengacu pada kenyataan bahwa beberapa submasalah sering muncul lagi dalam perhitungan yang berbeda, sehingga kita dapat menyimpan hasil perhitungan submasalah tersebut untuk menghindari perhitungan berulang.Prinsip Optimalitas: Unsur ini berhubungan dengan prinsip bahwa solusi optimal untuk masalah besar dapat ditemukan dengan menggabungkan solusi optimal dari submasalah yang lebih kecil. Dengan kata lain, dynamic programming mengasumsikan bahwa kita dapat memecahkan masalah besar menjadi masalah yang lebih kecil dan membangun solusi secara bertahap dari bawah ke atas.

Kedua unsur ini merupakan landasan dari teknik dynamic programming yang digunakan untuk memecahkan berbagai jenis masalah optimasi dan pemrograman, seperti penjadwalan, pencarian rute terpendek, atau masalah knapsack.

Pemrograman berorientasi objek merupakan paradigma pemrograman berdasarkan konsep "objek", yang dapat berisi data, dalam bentuk field atau dikenal juga sebagai atribut; serta kode, dalam bentuk fungsi/prosedur atau dikenal juga sebagai method. OOP (Object Oriented Programming) adalah suatu metode pemrograman yang berorientasi kepada objek. Tujuan dari OOP diciptakan adalah untuk mempermudah pengembangan program dengan cara mengikuti model yang telah ada di kehidupan sehari-hari.
